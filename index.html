Invalid Drug Promotion in COVID-19 / RStudio Project

Subject: Evaluating the economic impact of unfounded promotion of hydroxychloroquine during the pandemic.

Description: During the COVID-19 crisis, certain drugs like hydroxychloroquine and azithromycin were subject to misleading and unfounded communication. This project assesses the excess costs generated by this misinformation by analyzing French National Health Insurance (Assurance Maladie) reimbursements for these treatments. An interrupted time series analysis was conducted (ARIMA models with intervention) over the period from January 2015 to March 2022. View R Code

Methodology:
- Time series data (DRUGS-SAFEr database)
- ARIMA with intervention variable (February/March 2020)
- Before/after comparison and estimation of excess costs
- Negative control using anti-TNF drugs

Result: An excess reimbursement of over 1 million euros was estimated, representing approximately 12% above expected levels. No similar effect was observed for the control treatments.

Submission: Project submitted to the SFPT 2025 congress (abstract No. 00161). Read the abstract

Econometric Analysis of RIG Stock Returns – R/RStudio Project

Subject: Perform econometric analysis on the log returns of a financial asset and derive an optimal Volatility model (GARCH) for Risk Modeling (VaR).

Description: This project focused on studying the eight stylized facts of financial asset returns applied to RIG (Offshore Drilling) stock, using a database of 2,730 observations (2015-2025). The analysis confirmed that the series was stationary but exhibited massive leptokurtosis (heavy tails) and strong volatility clustering. These findings justified the use of an EGARCH model with a GHYP Distribution for errors, thereby confirming the existence of an asymmetric Leverage Effect.

Key Steps:
- Property Verification: Analysis of the 8 statistical facts (stationarity, skewness, kurtosis, seasonality, etc.).
- Linear Modeling: Identification and estimation of an ARMA model to capture weak linear dependence in returns.
- Volatility Modeling: Selection of an EGARCH(1,1) model to capture volatility clustering and leverage effects.
- Shock Distribution: Utilizing the GHYP Distribution (Generalized Hyperbolic) to model residual kurtosis and heavy tails.
- Seasonality Analysis: Examination of statistics by day of the week and month, revealing Strong Daily Seasonality (Monday being the most volatile).
- Backtesting & VaR: Value-at-Risk estimation and validation through coverage (Kupiec) and independence (Christoffersen) tests.

Project Files:
- Full Report & Appendices (PDF)
- R Code - Estimation Sample Analysis (RMarkdown)
- R Code - Test Sample Analysis (RMarkdown)
- R Code - GARCH Modeling & Backtesting (R Script)

Football Transfer Window Management – Excel VBA Project

Subject: Realistically simulate player transfer operations while respecting specific budget constraints for each professional football club.

Description: This project is a VBA application developed in Excel to mimic a transfer window management tool. The main objective was to model the complete mechanics of transfers and ensure rigorous financial tracking. The application manages a player database, allows entry of transfer offers, and automatically adjusts the budgets of buying and selling clubs in a way that faithfully aligns with real-world football financial logic.

Key Steps:
- Player Database: Built a database centralizing name, position, age, club, market value, and status (for sale, targeted, etc.). Populated via Excel import or UserForm.
- Transfer Management: Developed a "Make an Offer" UserForm to record transactions. The operation updates player affiliation and impacts budgets.
- Realism Rules: Integrated a constraint preventing offers lower than 90% of the player's market value.
- Budget Tracking: Created a "Club Budgets" sheet with initial budget, automatic calculation of sales/purchases/salaires, and a visual alert (red color) if the budget becomes negative.
- History & Search: Set up a transfer history system and a multi-criteria player search tool via UserForms.
